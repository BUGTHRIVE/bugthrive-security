# BugThrive Security

Official security policies, vulnerability disclosure guidelines, and resources for security researchers working with BugThrive.

---

> [!IMPORTANT]
> Do not report security vulnerabilities through public GitHub Issues, Discussions, pull requests, or any other public channel.

---

## Report a Vulnerability

GitHub Private Vulnerability Reporting is our recommended channel — it enables faster triage, coordinated handling, and direct communication with our security team.

**To submit a report:**

1. Navigate to the **Security** tab of this repository
2. Click **Report a vulnerability**
3. Complete and submit the private vulnerability report

**Alternatively**, reports may be submitted by email to `compliance@bugthrive.com` with the subject line: Security Vulnerability Report — [Vulnerability Title]

## Response Targets

| Stage | Target |
|---|---|
| Acknowledgement and first response | Within 2 business days |
| Initial assessment | Within 4 business days |
| Updates for accepted reports | Every 2 business days |
| Target final resolution | Within 7 business days |

> Business days exclude weekends and public holidays in India.

---

## Recognition

Researchers with validated reports will receive a **Letter of Appreciation** from BugThrive.

BugThrive does not currently operate a monetary bug bounty program. Submission or validation of a report does not create an entitlement to financial compensation or any other reward.

## Contact

For questions about this policy or scope clarifications, contact us at `compliance@bugthrive.com` before beginning any testing.
